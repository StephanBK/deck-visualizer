## Decisions made (Stephan, 2026-07-07/08)

1. **Phased approach:** generic animation first, parameterized by the customer's
   chosen deck material color. Later: Phase 2 = same animation over their real
   house photo; Phase 3 = fully AI-customized video (tech not reliable yet).
2. **iPad-only playback** — plays in the app during the sales conversation.
   No MP4 export / sharing needed for now.
3. **Placement:** Results screen, behind a "See how it's built" button.
4. **Style (v2 feedback):** must be visually impressive — isometric 3D-style
   scene, parts FLY IN and land in place (columns dropping in, joists
   rapid-firing, boards sweeping in the chosen color), camera moves per step,
   dusk + string-lights finale. v1 (flat 2D side-view) was rejected: too static,
   and the deck rendered too high on the house (proportions must read true —
   deck surface just below the patio-door threshold, columns to the ground).

## How the prototype works (port notes)

- Pure SVG generated by JS: an isometric projection helper (`px(x,y,z)`) +
  `box()` that emits top/right/front faces per cuboid. No libraries.
- Build sequence = 9 "beats" (Site, Layout, Ledger, Footings, Columns,
  Beam+joists, Decking, Railing, Dusk finale). Each beat is an SVG group
  `data-beat="n"`; pieces inside get class `pc` with staggered
  `transition-delay` (`--d`). Setting step N adds `.on` to groups 0..N —
  CSS transitions with an overshoot cubic-bezier make parts "fly in".
- Camera: CSS transform per beat on a `#camera` group (translate+scale, 1.9s).
- **Color engine:** deck boards + rail caps use `fill: var(--deck-face)` with
  `color-mix()` for the darker iso faces. Swapping one CSS custom property
  recolors everything — in the app, feed the selected catalog material's swatch
  color into `--deck-face`. All 66 materials work with zero extra assets.
- Captions + play/pause + tappable step dots (same interaction as the wizard).
- `prefers-reduced-motion` honored; meta charset + viewport required (learned
  the hard way).

## Construction accuracy

Steps and captions verified against research in
[deck-construction-research.md](deck-construction-research.md). Key corrections
already applied to the prototype:
- **Railing = 42" (California CRC §R312), not 36"** — caption and drawn
  geometry updated.
- Footings: min 12" into undisturbed soil (San Diego, no frost line).
- Ledger 1½–2″ below door threshold + flashing story.

Not yet in the prototype (candidates for the full version): stairs, knee
braces, hold-down hardware callouts, fascia/finish step, "final inspection"
closing beat, seismic/coastal talking points from the dossier.
